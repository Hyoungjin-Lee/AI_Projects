## Stage 9: 코드 리뷰 (Code Review)

**현재 단계**: Stage 9 - code_review
**담당**: Claude
**모델**: Sonnet (비판적 검토)
**노력**: High (15-30분, 20-40K 토큰)
**입력**: Codex Stage 8 구현 결과 (코드 + 테스트)
**산출물**: `docs/04_implementation/implementation_progress.md` (리뷰 로그)

---

### 목표

Codex의 구현 결과를 설계 기준으로 검토합니다. 기획/설계와 구현의 일치 여부, 구조적 문제, 예외 처리, 테스트 완성도를 평가합니다.

---

### 검증 기준

1. **설계와 구현 일치**: 기술 설계를 따랐는가?
2. **구조적 적절성**: 모듈 분리, 책임 분리가 명확한가?
3. **예외 처리**: 모든 오류 케이스를 다루었는가?
4. **테스트 커버리지**: 충분한가?
5. **보안**: 입력 검증, 민감정보 처리 올바른가?
6. **성능**: 성능 문제가 없는가?
7. **가독성**: 코드가 이해하기 쉬운가?
8. **유지보수성**: 다른 개발자가 쉽게 유지보수할 수 있는가?

---

### 작업 지침

1. **일치하는 부분**:
   - 설계 의도가 잘 구현된 부분
   - 좋은 패턴 사용 부분

2. **문제점**:
   - 설계와의 편차
   - 구조적 문제
   - 예외 처리 누락
   - 테스트 부족

3. **위험 요소**:
   - 성능 문제
   - 보안 문제
   - 유지보수 어려움
   - 버그 가능성

4. **수정 우선순위**:
   - Critical: 배포 불가
   - High: 즉시 수정 필요
   - Medium: 수정 권장
   - Low: 향후 리팩토링

5. **Codex에 넘길 수정 지시문**:
   - 명확하고 구체적인 지시
   - 필요시 코드 스니펫 제시

---

### 출력 형식

```
# Stage 9: 코드 리뷰

## 전체 평가
(구현 품질 및 설계 준수도 요약)

## 일치하는 부분
- (설계 따른 부분 1)
- (좋은 패턴 1)

## 문제점

### Critical (배포 불가)
1. (문제 1)
   - 위치: ...
   - 영향: ...
   - 수정 방법: ...

### High (즉시 수정)
1. (문제 1)

### Medium (권장)
1. (문제 1)

### Low (향후)
1. (문제 1)

## 위험 요소

| 요소 | 위험도 | 설명 |
|------|--------|------|
| ... | High | ... |

## 수정 지시문 (Codex용)

### Priority 1: Critical
1. (지시 1)
   \`\`\`python
   # 수정 전
   # 수정 후
   \`\`\`

2. (지시 2)

### Priority 2: High
(High 우선순위 항목)

## 다음 단계 권장
- [ ] revise (Stage 10) 필요시
- [ ] final_review (Stage 11) 진행 가능시
```

---

### 주의사항

- ✅ stockpilot 기존 패턴 확인 (Keychain, 로깅, 에러 처리)
- ✅ 보안 안티패턴 체크 (하드코딩 정보, 미검증 입력)
- ✅ 건설적이고 구체적인 피드백
- ❌ 코드 스타일만 지적하는 것 금지 (구조, 로직 중심)
- ✅ 명확한 수정 방법 제시
