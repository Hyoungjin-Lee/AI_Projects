## Stage 7: UI 플로우 설계 (UI Flow)

**현재 단계**: Stage 7 - ui_flow (선택)
**담당**: Claude
**모델**: Sonnet
**노력**: Medium (20-40분, 20-40K 토큰)
**입력**: Stage 6 UI 요구사항
**산출물**: `docs/03_design/ui_flow.md` + 와이어프레임

---

### 목표

UI 요구사항을 바탕으로 구체적인 화면 흐름, 상태 머신, 와이어프레임을 설계합니다.

---

### 주의사항

이 단계는 **선택 사항**입니다:
- **포함**: 웹/모바일 UI 설계 필요시
- **제외**: 백엔드 전용 시스템

---

### 포함 항목

1. **사용자 흐름 (User Flow)**:
   - 시작점 → 종료점의 경로
   - 각 화면에서의 선택지

2. **상태 머신**:
   - 화면 상태 전환
   - 로딩, 에러, 성공 상태

3. **와이어프레임**:
   - 텍스트/ASCII 형식 (또는 링크)
   - 레이아웃 및 요소 배치

4. **인터랙션 명세**:
   - 버튼 클릭 → 다음 상태
   - 입력 검증 규칙

---

### 출력 형식

```
# Stage 7: UI 플로우

## 사용자 흐름

(ASCII 다이어그램)

시작
  │
  ├─ 메인 대시보드 진입
  │  │
  │  ├─ 검색 버튼 클릭 → 검색 모달
  │  │  │
  │  │  └─ 종목 선택 → 상세 정보
  │  │
  │  └─ 설정 버튼 → 설정 화면
  │
  └─ 종료

## 상태 머신

```
INITIAL
  ├─ LOADING
  │  └─ READY
  │     ├─ SEARCHING
  │     │  └─ RESULTS
  │     │     └─ DETAIL
  │     └─ SETTINGS
  │        └─ READY
  └─ ERROR
     └─ RETRY
```

## 와이어프레임

### 메인 대시보드
```
┌──────────────────────────┐
│  Stockpilot  [⚙️ 설정]    │
├──────────────────────────┤
│ 🔍 검색                    │

│ AAPL    $150.23   +2.1% ▲ │
│ MSFT    $380.10   -1.2% ▼ │
│ GOOGL   $140.50   +0.5% ▲ │

└──────────────────────────┘
```

## 인터랙션 명세

| 요소 | 액션 | 결과 |
|------|------|------|
| 검색 버튼 | 클릭 | 검색 모달 오픈 |
| 종목 클릭 | 클릭 | 상세 정보 페이지로 이동 |
| 설정 버튼 | 클릭 | 설정 화면으로 이동 |
| 뒤로가기 | 클릭 | 이전 화면으로 돌아감 |
```
