## Stage 2: 계획 초안 (Planning Draft)

**현재 단계**: Stage 2 - planning_draft
**담당**: Claude
**모델**: Sonnet (반복 속도 중요, 충분한 깊이)
**노력**: Medium (15-30분, 15-30K 토큰)
**입력**: Stage 1 브레인스토밍 결과
**산출물**: `docs/02_planning/plan_draft.md`

---

### 목표

Stage 1의 브레인스토밍 결과를 바탕으로 구체적인 기획 초안을 작성합니다. 문제 정의, 사용자 흐름, 핵심 기능, 리스크를 명확히 정의합니다.

---

### 작업 지침

1. **프로젝트 개요**:
   - 한 문단으로 프로젝트 요약
   - "이 프로젝트의 목표는 ... 이다"로 시작

2. **목표 정의**:
   - 주요 목표 (Primary Goal)
   - 부차 목표 (Secondary Goals)
   - 성공 기준 (Success Criteria)

3. **사용자 문제 정의**:
   - "사용자가 해결하려는 구체적인 문제는?"
   - 현재 상황 vs 원하는 상황
   - stockpilot 컨텍스트: 평일 4회 브리핑, 자동화 시스템

4. **핵심 기능** (우선순위 포함):
   - 반드시 포함될 기능 (Must-Have)
   - 향후 개선 기능 (Nice-to-Have)

5. **제외 범위**:
   - 명시적으로 하지 않을 것들
   - "이 프로젝트는 X를 포함하지 않습니다"

6. **예상 사용자 흐름**:
   - 주요 사용자 여정 (User Journey)
   - 예: "사용자가 아침 8시에 카카오톡 메시지 수신 → 주식 정보 확인 → 설정 변경"

7. **기술적 주의사항**:
   - stockpilot 기존 구조 (KIS API, keychain, launchd)
   - 성능 고려사항
   - 외부 의존성

8. **리스크 & 완화 방법**:
   - 기술적 리스크
   - 운영 리스크
   - 각 리스크별 완화 방법

- ❌ 기술 구현 세부사항 포함 금지 (아직 Stage 5)
