## Stage 1: 아이디어 구상 (Brainstorm)

**현재 단계**: Stage 1 - brainstorm
**담당**: Claude
**모델**: Opus (추론 깊이 필요, 창의적 문제 분석)
**노력**: Medium (15-30분, 15-30K 토큰)
**입력**: 사용자 요청, 요구사항
**산출물**: `docs/01_brainstorm/brainstorm.md`

---

### 목표

프로젝트 방향성을 탐색하고, 가능한 접근 방식들을 비교한 후, 현실적으로 실행 가능한 최선의 방향을 추천합니다.

---

### 작업 지침

1. **문제 정의**: 사용자 요청의 핵심 문제를 명확히 하세요
   - 구체적인 표현 사용
   - 기술적 제약이 없다고 가정
   - 사용자의 실제 니즈 파악

2. **가능한 방향 3개 이상 제시**:
   - 각 방향은 서로 다른 접근 방식 (예: API 기반 vs 직접 구현, 마이크로서비스 vs 모놀리식)
   - 현실적인 범위 내에서

3. **각 방향의 장점 & 단점 분석**:
   - 개발 시간
   - 확장성
   - 유지보수성
   - 기술적 위험
   - stockpilot 기존 구조와의 호환성

4. **가장 현실적인 추천안**:
   - 이유를 명확하게 설명
   - 팀의 기술 스택 고려 (Python, KIS API, 카카오톡/텔레그램)
   - 비용-편익 분석

5. **다음 단계에서 결정할 질문**:
   - 계획 단계에서 탐색해야 할 항목들

---

### 출력 형식

```
# Stage 1: 아이디어 구상

## 문제 정의
(명확하고 구체적인 문제 진술)

## 가능한 방향
### 방향 A: (이름)
**설명**: ...
**장점**:
- ...
**단점**:
- ...

### 방향 B: (이름)
(동일한 구조)

### 방향 C: (이름)
(동일한 구조)

## 추천안: 방향 X
(선택 이유, 장기적 가치, 현실성)

## 다음 단계에서 결정할 사항
1. ...
2. ...

## 제약 & 가정
- (기술 제약)
- (시간 제약)
- (팀 능력)
```

---

### 주의사항

- ❌ 기술 구현 세부사항 포함 금지 (아직 Stage 2)
- ❌ "가능한 모든 방향" 나열 금지 (3-4개로 충분)
- ✅ stockpilot 기존 구조 고려 (Python, KIS API, Keychain 관리, launchd 스케줄)
- ✅ 사용자 피드백 기준 변경 가능
- ✅ 모호한 부분은 명시적으로 질문
