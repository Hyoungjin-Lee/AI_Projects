## Stage 12: QA & 릴리스 (QA & Release)

**현재 단계**: Stage 12 - qa
**담당**: Claude
**모델**: Sonnet
**노력**: Medium (15-30분, 15-25K 토큰)
**입력**: 최종 검증된 코드
**산출물**: `docs/05_qa/qa_scenarios.md` + `release_checklist.md`

---

### 목표

실제 배포 전에 QA 시나리오와 체크리스트를 작성합니다. 테스트 범위, 우선순위, 완료 판단 기준을 명확히 합니다.

---

### 작업 지침

1. **정상 흐름 테스트**:
   - 기획한 모든 기능의 Happy Path
   - 예: "아침 8시 리포트 정상 수신"

2. **경계값 테스트**:
   - 최소/최대값
   - 공백, 특수문자
   - 예: "종목 코드 1자 입력", "10,000자 텍스트"

3. **예외 상황 테스트**:
   - 네트워크 끊김
   - API 타임아웃
   - 파일 없음
   - 권한 문제

4. **실패 복구 테스트**:
   - 실패 후 재시도
   - 로그 기록 확인
   - 시스템 복구 가능성

5. **로그/에러 확인 포인트**:
   - 어디를 체크할 것인가?
   - `logs/` 파일 확인
   - 에러 메시지 명확성

6. **우선순위별 테스트**:
   - P0: Must-Pass (배포 조건)
   - P1: Should-Pass (배포 권장)
   - P2: Nice-to-Pass (차기 버전)

7. **배포 전 체크리스트**:
   - 모든 테스트 통과?
   - 문서 업데이트?
   - 버전 번호 업데이트?
   - 롤백 계획?

---

### 출력 형식

```
# Stage 12: QA & 릴리스

## QA 범위

### 정상 흐름 테스트 (Happy Path)
1. **아침 리포트 (08:30)**
   - 시간: 08:30 정확히 또는 ±2분 내
   - 수신: 카카오톡 메시지 도착
   - 내용: 관심종목 5개 분석 결과 포함

2. **마감 리포트 (20:30)**
   - (동일)

### 경계값 테스트
1. **종목 코드 입력**
   - 입력: 유효한 코드 (예: "005930")
   - 입력: 무효한 코드 (예: "invalid")
   - 결과: 에러 메시지 명확

2. **시간 설정**
   - 입력: 유효한 시간 (08:30)
   - 입력: 자정 (00:00), 23:59
   - 입력: 음수, 25시간

### 예외 상황 테스트
1. **네트워크 끊김**
   - 상황: KIS API 응답 없음
   - 예상: 재시도 3회 후 에러 로그
   - 확인: `logs/morning_report.log` 에러 메시지

2. **파일 손상**
   - 상황: watchlist.json 손상
   - 예상: 자동 복구 또는 명확한 에러

### 실패 복구 테스트
1. **API 타임아웃 → 재시도**
   - 1차 실패
   - 2초 대기
   - 재시도 성공 여부

## 우선순위별 테스트

### P0: Must-Pass (배포 조건)
- [ ] 아침 리포트 정상 전송
- [ ] 마감 리포트 정상 전송
- [ ] 에러 발생 시 로그 기록
- [ ] 카카오톡 API 인증 작동

### P1: Should-Pass
- [ ] 관심종목 추가/삭제 작동
- [ ] 기술적 분석 지표 정확성
- [ ] 성능: 리포트 생성 < 10초

### P2: Nice-to-Pass
- [ ] 텔레그램 봇 연동 (향후)
- [ ] 웹 대시보드 (향후)

## 배포 전 체크리스트

### 코드 확인
- [ ] 모든 P0 테스트 통과
- [ ] 모든 P1 테스트 통과 (권장)
- [ ] 성능 벤치마크 완료
- [ ] 메모리 누수 확인 (장시간 실행 테스트)
- [ ] 로그 레벨 확인 (DEBUG 제거)
- [ ] API 키/비밀번호 노출 여부 확인

### 문서 확인
- [ ] README 업데이트
- [ ] API 문서 최신화
- [ ] CHANGELOG 작성
- [ ] 운영 가이드 작성

### 시스템 확인
- [ ] Keychain에 모든 정보 저장
- [ ] launchd plist 파일 업데이트
- [ ] 로그 파일 디렉토리 확인
- [ ] 권한 설정 확인

### 운영 확인
- [ ] 모니터링 알림 설정
- [ ] 에러 처리 계획 수립
- [ ] 롤백 계획 수립
- [ ] 배포 후 검증 계획

## 완료 판단 기준

### 배포 Go
- 모든 P0 테스트 ✓
- 치명적 문제 없음 ✓
- 문서 완료 ✓

### 배포 No-Go
- P0 테스트 실패
- 성능 문제 (임계값 초과)
- 보안 문제
```
