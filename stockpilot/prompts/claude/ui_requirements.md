## Stage 6: UI 요구사항 (UI Requirements)

**현재 단계**: Stage 6 - ui_requirements (선택)
**담당**: Claude
**모델**: Sonnet
**노력**: Medium (15-30분, 15-30K 토큰)
**입력**: Stage 5 기술 설계 (프론트엔드 필요시)
**산출물**: `docs/03_design/ui_requirements.md`

---

### 목표

프론트엔드가 필요한 프로젝트일 경우, 사용자 관점의 UI 요구사항을 정의합니다.

---

### 주의사항

이 단계는 **선택 사항**입니다:
- **포함**: 웹/모바일 UI가 필요한 경우
- **제외**: 백엔드 전용 시스템 (stockpilot의 대부분)

---

### 포함 항목

1. **사용자 스토리**:
   - "사용자로서 X를 하려고 할 때, Y가 가능해야 한다"
   - Acceptance Criteria 포함

2. **화면 목록**:
   - 각 화면의 역할
   - 화면 간 내비게이션

3. **주요 상호작용**:
   - 사용자 입력 (클릭, 입력 등)
   - 응답 (피드백, 결과)

4. **접근성 요구사항**:
   - 키보드 네비게이션
   - 스크린 리더 지원

---

### 출력 형식

```
# Stage 6: UI 요구사항

## 사용자 스토리
### Story 1: 주식 검색
**As a** 사용자
**I want to** 특정 주식을 검색할 수 있기를
**So that** 관심 종목을 추가할 수 있다

**Acceptance Criteria**:
- [ ] 검색창에 텍스트 입력 가능
- [ ] 결과 목록 표시
- [ ] 결과 클릭 시 상세 정보 표시

## 화면 목록
1. 메인 대시보드
   - 보유 종목 목록
   - 오늘의 변화율

2. 종목 상세
   - 차트
   - 기술적 분석 결과

3. 설정
   - 알림 설정
   - 관심 종목 관리

## 주요 상호작용
(flow 설명)

## 접근성
- 모든 입력은 키보드로 가능
- 색상만으로 정보 전달 금지
```
