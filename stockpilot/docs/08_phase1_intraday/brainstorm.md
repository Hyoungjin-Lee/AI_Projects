## 1. 배경 및 동기

`intraday_discovery.py`는 v2.1(2026-04-19)에 구현되어 실제 운영 중.
현재까지의 발굴 성과:
- 이수페타시스: 발굴 +7.2% → 장마감 +13.13% (+5.9%p)
- 에코프로머티리얼즈: 발굴 +5.4% → 장마감 +7.06%

그러나 **발굴 조건이 얼마나 정확한지 데이터가 없고**, 필터 기준도 운영 경험 없이 설정된 상태.
Phase 1 목표: 발굴 정확도를 높이고, 성과를 추적해서 기준값 개선의 근거를 만드는 것.

---

## 2. 현재 구조 요약

```
R1 수집 (09:03) → R2 수집 (09:05) → 교집합 → 이격도 과열제외 → 점수산정 → top3 전송
```

### 현재 점수 체계 (v2.4 기준, 오늘 수정 반영)
- 체결강도 `pow_2 >= 110` 필수 (미달 시 탈락) ← 오늘 추가
- 등락률 `flc_2 >= 2.0%` 필수 (미달 시 탈락) ← 오늘 추가
- 체결강도 130↑: +3 / 120↑: +2 / 110↑: +1 ← 오늘 수정
- 등락률 R1→R2 상승: +1
- 거래량 R1→R2 증가: +1
- HTS 상위 10위: +1
- 최대 점수: 6점

## 3. 문제 정의

### 문제 1 — 발굴 성과 추적 불가
- 매일 발굴하지만 "얼마나 맞았는지" 기록이 없음
- 기준값(체결강도 110, 등락률 2%) 조정 근거가 없음
- 개선 방향: **발굴 성과 추적 DB** 구축

### 문제 2 — 텔레그램 메시지 정보 부족
- 현재 top3만 표시, 4~5위 후보는 안 보임
- 발굴 종목의 "왜 발굴됐는지" 이유가 불충분
- 개선 방향: **후보 확장 표시 + 발굴 이유 강화**

## 4. 아이디어 목록

### A. 발굴 성과 추적 DB (핵심)

**개념:**
```
발굴 시점 기록 → 당일 종가 자동 업데이트 → 수익률 계산
```

**저장 위치:** `data/discovery_log.json`

**저장 항목:**
- 날짜, 발굴시각, 종목코드, 종목명
- 발굴 시점 가격 (`disc_price`)
- 발굴 시점 체결강도, 등락률, 점수
- 당일 종가 (closing_report 실행 시 자동 업데이트)
- 수익률 = (종가 - 발굴가) / 발굴가 × 100

**활용:**
- 주간/월간 발굴 정확도 통계
- 체결강도 기준별 수익률 분포 → 기준값 근거 확보

### B. 텔레그램 메시지 개선

**개념:**
- 후보 3개 초과 시 4~5위도 "관심 후보"로 추가 표시
- 발굴 이유 텍스트 강화 (체결강도 구간 설명 추가)

**예시:**
```
🥇 이수페타시스 (080010)  6점
   체결강도: 145 (+12↑) | 등락률: +4.2%↑ | 거래량↑
   🌐 온라인 관심 2위

───────────
📋 추가 관심 후보
  4위 삼성전자 (005930) — 체결강도: 118 | 등락률: +2.1%
  5위 SK하이닉스 (000660) — 체결강도: 112 | 등락률: +2.8%
```

### C. 발굴 성과 요약 메시지 (선택)

전날 발굴 종목의 결과를 모닝 리포트에 한 줄 추가:
```
📊 어제 발굴 성과: 이수페타시스 +5.9%p ↑ | 에코프로 +1.6%p ↑
```

## 6. 기술적 제약

- `data/discovery_log.json` — 날짜별 append 방식, 최대 30일치 보관
- 종가 업데이트는 `closing_report.py` 실행 시 사이드이펙트로 처리
  - closing_report가 이미 보유종목 일봉 데이터를 가져오므로, 발굴 종목도 같은 API 호출로 처리 가능
- 성과 추적은 read-only 용도 (매매 신호로 사용 X)
