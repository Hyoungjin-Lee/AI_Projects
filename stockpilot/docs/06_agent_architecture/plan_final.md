# Stage 2~4: 에이전트 아키텍처 개선 계획

> 날짜: 2026-04-19 | 담당: Claude Sonnet | Effort: Medium
> 기존 소스 유지 + 신규 기능 추가 방식

---

## 프로젝트 개요

stockpilot v1.0.0의 단방향/독립 실행 구조를 **협업하는 에이전트 팀**으로 진화시킨다.
기존 5개 스크립트는 그대로 유지하고, 세 가지 기능을 추가한다.

---

## 개선 목표 3가지

### Goal 1: 실행 에이전트 공유 상태 (daily_state.json)

**문제:** 5개 스크립트가 독립 실행 → 당일 컨텍스트 공유 불가
**해결:** 공유 상태 파일로 에이전트 간 소통 채널 구축

**추가 파일:**
- `data/daily_state.json` — 당일 공유 상태
- `morning_report/state_manager.py` — 상태 읽기/쓰기 공통 모듈

**각 스크립트 역할:**
| 스크립트 | 읽기 | 쓰기 |
|----------|------|------|
| morning_report | — | 시장 방향, 보유종목 시그널 |
| intraday_report | 시장 방향 | 장초기 알림 |
| closing_report | 전체 상태 | 최종 결산 요약 |
| stock_discovery | 시장 방향, 시그널 | 발굴 결과 |
| watchlist_sync | — | 관심종목 변경 여부 |

---

### Goal 2: 텔레그램 오케스트레이터 (telegram_bot.py)

**문제:** 텔레그램 단방향 — 명령 수신 불가
**해결:** 상시 실행 봇 데몬 + 오케스트레이터 구조

**추가 파일:**
- `morning_report/telegram_bot.py` — 봇 데몬 (polling)
- `morning_report/orchestrator.py` — 명령 라우팅

**Phase 1 명령어 (이번 구현):**
| 명령 | 동작 |
|------|------|
| `/잔고` | KIS 잔고 즉시 조회 |
| `/상태` | daily_state.json 요약 |
| `/발굴` | stock_discovery 즉시 실행 |
| `/도움말` | 명령어 목록 |

**Phase 2 명령어 (추후 별도 프로젝트):**
- `/매수`, `/매도` — 안전장치 설계 후 별도 진행

**launchd 추가:**
- `com.aigeenya.stockbot.plist` — 봇 데몬 상시 실행

---

### Goal 3: 워크플로우 독립 검증 프로토콜

**문제:** Stage 11 최종 검증이 같은 컨텍스트에서 진행 → 편향 가능
**해결:** WORKFLOW.md에 독립 검증 프로토콜 섹션 추가

**독립 검증 규칙:**
1. Stage 11 시작 시 **새 대화창**에서 진행
2. 이전 컨텍스트 전달 금지 — 코드와 요구사항만 전달
3. 검증관은 독립적 시각으로 보안/로직/엣지케이스 검토
4. 판정 결과를 `docs/` 에 저장 후 원래 세션으로 복귀
