## 3. 성공률 향상을 위한 13가지 신규 아이디어

### A. 시그널 신뢰도 강화

**A1. Confidence Cascade (다중 시간대 합의)**
- 단일 시간대 검출 = 약신호
- 2개 시간대(예: 1H + 15m) 동시 검출 = 표준
- 3개 시간대 동시 = 강신호
- 시그널 강도별 후속 액션 차등화

**A2. Convergence Score (지표 + 패턴 합의)**
- 기존 B+C 조합(SMA, RSI) 점수와 패턴 점수를 0~1로 정규화 후 가중 합산
- 양자 합의 시 진입, 한쪽만 시그널 시 보류, 충돌 시 회피
- 충돌 케이스가 가장 위험 — 명시적 룰 필요

**A3. Order Flow Confirmation (호가/체결강도 통합)**
- 비주얼 패턴 + 호가창 행동 일치 = 신뢰도 가중치 +
- 예: 바닥패턴 형성 + 매수 호가 누적 + 체결강도 100 초과 = 진짜 바닥
- KIS API 호가 10단계 + 체결강도 호출 권한 확인 필요 [확인 필요]

### B. 한국 시장 특화 필터

**B1. 외인/기관 수급 게이트**
- 패턴 시그널 + 직전 5일 외인+기관 순매수 → 신뢰도 ↑
- 패턴 + 개인 매수 우세 → 신뢰도 ↓ (반대 시그널 가능성)
- 한국 주식만의 차별 무기

**B2. 시간대별 가중치 (Time-of-Day Filter)**
- 09:00~09:30 시초가 변동 구간: 패턴 검출 보류
- 11:30~13:00 점심 거래 부진: 신뢰도 -
- 14:30 이후 동시호가 직전: 패턴 신뢰도 ↓ (마감 변동성)
- 가장 신뢰도 높은 구간: 10:00~11:30, 13:00~14:30

**B3. 시장 레짐 필터 (Regime-Aware Activation)**
- ADX/변동성 기반 추세장/횡보장/하락장 분류
- 추세장: 라인/공급 활성화
- 횡보장: 바닥/뚜껑 활성화
- 하락장: **신규 매수 전면 정지**, 서치 필터만 가동

### C. 자본 운용 강화

**C1. 시그널 강도별 포지션 사이징 (Kelly-Lite)**
- 약신호 = base × 0.5
- 표준 = base × 1.0
- 강신호(다중 확인) = base × 1.5
- 기존 `split_weights: [0.5, 0.3, 0.2]` 분할매수 구조와 결합

**C2. Drawdown-Adaptive Aggression**
- 누적 손익 기반 공격성 자동 조절
- 누적 -5% → 강신호만 거래
- 누적 -10% → 신규 진입 정지, 청산만
- 누적 +10% → 표준 복귀
- 인간 본능(연속 손실 후 무모해짐) 차단

**C3. A/B 실험 레인 (10% 실험 자본)**
- 자본의 10%는 항상 "신규 패턴 실험 레인"
- 검증 안 된 패턴은 여기서만 작동
- N건 표본 누적 + 승률 검증 후 메인 레인 승격

### D. 학습 및 자동 개선

**D1. 패턴 라이프사이클 트래킹**
- 검출된 모든 패턴(거래 여부 무관)의 후속 24h/72h 가격 자동 기록
- 일정 표본(예: 100건) 누적 후 패턴별 실측 승률/RR 산출
- 임계값 자동 조정 (rolling)

**D2. Anti-Pattern 라이브러리**
- 검출했지만 실패한 패턴 별도 저장
- 시간 누적 시 false positive 카탈로그 형성
- 검출 시 anti-pattern과 유사도 비교 → 신뢰도 차감

**D3. 이중 검증 게이트 (Two-Person Rule)**
- 자동 매수 전 (1) 알고리즘 시그널 + (2) AI(LLM) 차트 해석 두 번 통과 시에만 발주
- AI 호출 비용 발생하지만 false positive 대폭 감소
- stockpilot은 이미 AI 기반 — 자연스러운 확장

### E. 운영 안전장치

**E1. 단계적 롤아웃 (Shadow → Alert → Trade)**
- Shadow: 검출만 하고 로그
- Alert: 텔레그램 알림만 발송, 거래 X
- Trade-Small: 최소 단위로 실주문 (1주 등)
- Trade-Full: 정상 사이즈
- 단계별 N건 + 승률 임계값 통과 후만 다음 단계 승격

**E2. Kill Switch (긴급 정지)**
- 일중 누적 손실 -3% 도달 → 패턴 매매 전면 정지
- 패턴 false positive 율 30% 초과 (직전 20건 기준) → 자동 비활성화
- 텔레그램 즉시 알림

## 5. 단계별 로드맵 (Phased Rollout)

### Phase A — 인프라 구축 (2주)
1. 백테스트 환경 구축/검증
2. 분봉/호가 데이터 권한 검증
3. 라인 자동 검출기 PoC
4. HA 변환 함수
5. 패턴 라이프사이클 트래커 골격

**Exit 기준**: 백테스트로 임의 종목 1년 데이터 라인 검출 + 시각 검증 통과

### Phase B — 서치 필터 도입 (1~2주)
1. 서치 패턴 검출 모듈
2. **진입 거부권**으로 통합 (B+C 충족이어도 서치 감지 시 보류)
3. Shadow 모드 운영 → 7일 통계 확인
4. Alert 모드 → 7일 통계 확인
5. Trade-Small → 본격 적용

**Exit 기준**: 진입 거부권 활성화 종목의 후속 5일 평균 수익률이 미활성 종목 대비 유의미하게 낮은지 확인

### Phase C — 공급/라인 진입 보조 (3주)
1. 공급/라인 패턴 검출
2. B+C 조합과 Convergence Score 통합
3. 시그널 강도별 포지션 사이징
4. 시장 레짐 필터 적용
5. 단계적 롤아웃

**Exit 기준**: 백테스트 + 실거래 표본 50건 이상에서 기존 B+C 단독 대비 승률 또는 R:R 개선

### Phase D — 확장 (지속)
1. 바닥/뚜껑 패턴 (정책 결정 후)
2. AI 차트 해석 이중 검증 게이트
3. 외인/기관 수급 게이트
4. Anti-Pattern 라이브러리 자동 학습
