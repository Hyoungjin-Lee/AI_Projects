# Stage 1: 브레인스토밍 — 카카오톡 → 텔레그램 전환

> 날짜: 2026-04-18 | 담당: Claude Opus | Effort: Medium

---

## 문제 정의

카카오톡 "나에게 보내기"는 알림이 오지 않는 구조적 한계가 있다.
자동 브리핑 시스템의 핵심 가치는 "제때 알림을 받는 것"인데, 이 기능이 동작하지 않음.

**현재 상황:**
- `kakao_sender.py`가 5개 스크립트에서 `send_report()` / `send_text()` 형태로 호출됨
- 호출 스크립트: `morning_report.py`, `closing_report.py`, `intraday_report.py`, `stock_discovery.py`, `setup_kakao.py`
- 인증정보는 macOS Keychain에 저장 (보안 구조 완비)

---

## 전환 방향 3가지

### A. 텔레그램 봇 (추천)
- 봇 토큰 하나로 설정 완료, 알림 정상 작동
- 단방향(지금) → 양방향 명령(나중에) 확장 가능
- 무료, 서버 불필요, API 안정적

### B. 카카오 비즈니스 채널
- 알림은 오지만 설정 복잡 + 비용 발생 가능
- 현재 구조 유지 가능하지만 근본 해결책 아님

### C. 다른 메신저 (Slack, Discord 등)
- 주식 알림 목적엔 과도함
- 개인 사용에 텔레그램이 더 적합

---

## 추천안: 텔레그램 봇 (A)

**이유:**
1. 알림 문제 근본 해결
2. `kakao_sender.py` → `telegram_sender.py` 1:1 교체 구조
3. 기존 5개 스크립트 호출부 최소 수정 (함수 시그니처 동일하게 유지)
4. Keychain 보안 구조 그대로 재활용
5. 향후 `/매수`, `/잔고` 같은 양방향 명령 확장 기반
