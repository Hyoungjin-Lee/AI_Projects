# KIS API 자주 보는 에러 코드

API 호출에 실패하면 응답 JSON에 `rt_cd`, `msg_cd`, `msg1` 필드가 들어 있다. `rt_cd`가 `'0'`이면 성공, 그 외는 실패.

## 인증/토큰

| msg_cd | 뜻 | 대처 |
|---|---|---|
| `EGW00121` | 토큰 만료 | `data/cache/kis_token.json` 삭제 후 재시도. 클라이언트가 자동 처리하지만 만료 직전 호출 시 가끔 발생 |
| `EGW00123` | 유효하지 않은 토큰 | 위와 동일 |
| `EGW00133` | 1분당 토큰 발급 한도 초과 | 1분 대기. 캐시를 너무 자주 비웠을 때 |
| `EGW00201` | appkey/appsecret 불일치 | `.env` 값 확인. 모의투자용 키를 실전에 쓰지 않았는지 확인 |

## 시세

| msg_cd | 뜻 | 대처 |
|---|---|---|
| `MCA01070` | 종목코드 오류 | 6자리 숫자인지, 상장폐지/거래정지 종목 아닌지 확인 |
| `MCA01000` | 데이터 없음 | 비영업일 직후, 신규상장 직후 등 |

## 주문

| msg_cd | 뜻 | 대처 |
|---|---|---|
| `40240000` | 주문가능금액 부족 | 매수 전에 `inquire-psbl-order`로 가능 금액 확인 |
| `40310000` | 주문수량 오류 | 호가 단위 위반. ETF/저가주는 1주 단위지만 일부는 다름 |
| `40580000` | 장 시간 외 주문 | 일반 주문은 09:00~15:30. 시간외는 별도 ORD_DVSN |
| `41010000` | 단일가 시간대 | 동시호가(08:30~09:00, 15:20~15:30) 중에는 일부 주문 불가 |

## HTTP 레벨

- `403 Forbidden` — IP가 KIS 등록 IP와 다름. 외부 호스팅에서 호출할 때 자주 발생. KIS 포털에서 IP 등록 필요.
- `429 Too Many Requests` — 레이트 리밋 초과. 클라이언트의 `_throttle()`이 막아주지만 여러 프로세스가 동시 호출하면 터질 수 있음.

## 일반 디버깅 순서

1. 응답 JSON 전체를 stderr에 찍어서 본다 (`KISAPIError`가 던지는 메시지)
2. `msg_cd`로 위 표 확인
3. 그래도 모르겠으면 KIS 공식 포털 https://apiportal.koreainvestment.com 검색
