# stock-analysis: 주식 분석 스킬

## 이 스킬이 하는 일

`kis-api`가 가져온 OHLCV 데이터를 받아 다음 세 관점에서 종목을 분석한다.

1. **단타(데이트레이딩)** — 분봉/호가/거래량 급변·VWAP·당일 등락폭
2. **스윙(며칠~몇 주)** — 일봉 기반 이동평균·MACD·RSI·볼린저밴드·지지저항
3. **정량(시스템)** — 시그널 함수, 간단 백테스트, 리스크 메트릭(MDD, 변동성, 샤프)

각 관점은 독립적이지만, 사용자가 한 종목에 대해 "어때?"라고 물으면 세 관점을 모두 돌려 종합 판단을 보여주는 게 기본 동작이다. 사용자가 명시적으로 "단타로만 봐줘" 같이 좁히면 그 관점만 쓴다.

## 분석을 시작할 때 — 데이터 확보 순서

이 스킬은 데이터를 직접 가져오지 않는다. `kis-api` 스킬의 스크립트로 먼저 데이터를 받는다.

| 분석 관점 | 필요 데이터 | 호출 |
|---|---|---|
| 단타 | 분봉 30~60개 + 호가 + 현재가 | `get_minute_chart.py`, `get_orderbook.py`, `get_quote.py` |
| 스윙 | 일봉 60~120일 + 현재가 | `get_daily_chart.py --days 120`, `get_quote.py` |
| 정량 | 일봉 120~250일 | `get_daily_chart.py --days 200` |

데이터가 `data/raw/<코드>_<지표>_<날짜>.json`에 이미 캐시돼 있으면 같은 날짜의 파일은 재호출하지 않고 그대로 읽는다. 여러 번 분석할 때 KIS API 호출량을 아낄 수 있다.

## 핵심 라이브러리: indicators.py

기술적 지표는 모두 `scripts/indicators.py`에 함수로 정의돼 있다. pandas/numpy를 이미 import해서 쓰므로, 직접 인라인으로 다시 짜지 말고 이 모듈을 import해서 쓴다.

```python
from indicators import (
    sma, ema, rsi, macd, bollinger,
    atr, vwap, support_resistance,
    drawdown, sharpe,
)
```

함수는 모두 pandas Series를 받아 pandas Series(또는 dict)를 돌려준다. 원본 OHLCV는 `pd.DataFrame`으로 만들어 사용한다 (`scripts/loader.py`에 헬퍼 있음).

## 사용 가능한 분석 스크립트

| 스크립트 | 용도 |
|---|---|
| `analyze_swing.py <code>` | 스윙 분석 — MA20/60, RSI(14), MACD, 볼린저, 지지저항, 추세 판정 |
| `analyze_intraday.py <code>` | 단타 분석 — VWAP, 5분봉 추세, 호가 불균형, 당일 등락폭 |
| `analyze_quant.py <code>` | 정량 — 모멘텀(20일/60일), 변동성, MDD, 골든/데드크로스 시그널 |
| `analyze_full.py <code>` | 위 셋을 모두 돌려 종합 판단 (사용자가 "어때?"라고 물을 때 기본값) |

각 스크립트는 결과를 stdout에 JSON으로 찍는다. 형식은 다음과 같이 통일돼 있다:

```json
{
  "code": "005930",
  "perspective": "swing",
  "verdict": "BUY | HOLD | SELL | WATCH",
  "confidence": 0.62,
  "key_signals": [
    {"name": "RSI(14)", "value": 58.3, "interpretation": "중립~상승"},
    {"name": "MACD", "value": "골든크로스 임박", "interpretation": "긍정"}
  ],
  "support_levels": [69500, 67800],
  "resistance_levels": [73200, 75000],
  "narrative": "한 문장~한 단락 자연어 요약",
  "risks": ["거래대금 평균 대비 30% 감소", "코스피 약세"],
  "data_window": {"from": "2026-02-15", "to": "2026-04-17", "days": 60}
}
```

이 형식은 `trading-report` 스킬이 그대로 받아 리포트로 만들 수 있게 표준화한 것이다.

## 판정(verdict) 가이드

- **BUY** — 다중 시그널이 같은 방향, 진입 시 손절가까지 거리가 짧고 목표가까지 여유가 있음 (RR ≥ 2)
- **HOLD** — 보유 중이라면 유지, 신규 진입은 보류
- **SELL** — 보유 중이라면 청산 신호. 추세 반전 또는 손절 라인 근접
- **WATCH** — 시그널이 충돌하거나 데이터 부족. 진입하지 말고 며칠 더 관찰

`confidence`는 0~1 범위. 시그널 일치도와 거래대금/추세 명확도를 가중평균한 값. 이 값이 0.4 미만이면 verdict와 무관하게 항상 사용자에게 "확신도가 낮으니 참고만 하세요"라는 경고를 함께 보고한다.

## 분석할 때 항상 지키는 원칙

1. **수치만 던지지 않는다.** RSI 70은 "과매수"라는 사전적 의미로만 보지 말고, 추세·거래량과 합쳐 해석한다. 강한 상승추세 중의 RSI 70은 보통 과매수가 아니다.
2. **반대 시그널을 같이 본다.** "이 종목 사도 될까?"라는 질문에는 매수 근거 2~3개 + 반대(매도/관망) 근거 1~2개를 같이 제시한다. 한쪽만 강조하면 사용자에게 손해다.
3. **거래량 없는 신호는 약하다.** 거래대금이 20일 평균의 50% 미만이면 어떤 차트 신호도 신뢰도를 한 단계 낮춘다.
4. **개별 종목 분석에 시장 컨텍스트를 1줄 곁들인다.** 코스피/코스닥 지수, 환율, 미장 야간 흐름 같은 거시 환경. 사용자가 시장 상황을 모르면 종목 판단이 왜곡된다.
5. **백테스트는 슬리피지/세금/수수료를 반드시 포함한다.** `analyze_quant.py`의 백테스트 함수는 매도세 0.18%, 슬리피지 0.05%를 기본으로 차감한다. 무비용 백테스트는 거짓말이라는 걸 사용자에게도 알려준다.
